### Общая информация о задании
*   **Компетенция:** Разработка мобильных приложений.
*   **Мероприятие:** Региональный этап чемпионата «Профессионалы» (2026 год).
*   **Длительность:** 18 часов (3 конкурсных дня).
*   **Общая структура:** 7 модулей (5 инвариантных/обязательных и 2 вариативных).

### Технические требования и стек
1.  **Минимальные версии ОС:** Android 13.0, iOS 17.0.
2.  **Бэкенд:** SAAS решение **Pocketbase** (доступ только через REST API).
3.  **Система контроля версий:** Git (предоставляется организатором).
4.  **Управление задачами:** Используется таск-трекер (Kanban-доска).
    *   Статусы: *Новая* -> *В процессе* (не более 2 задач одновременно) -> *Можно проверять* (нужно указать коммит) -> *Завершена*.
5.  **Дизайн:** Строгое следование макетам в **Figma**. Вёрстка должна быть адаптивной (под разные экраны).
    *   Ссылка на Figma: [https://www.figma.com/design/pbwY6r5TtWUkxMsoJBWf0c/Matule-2026...](https://www.figma.com/design/pbwY6r5TtWUkxMsoJBWf0c/Matule-2026?node-id=0-1&t=IQsIknZ1tZSt3fFa-1)
6.  **API Документация (Swagger):** [https://raw.githubusercontent.com/serk87/saransk2025/final20225/api.yaml](https://raw.githubusercontent.com/serk87/saransk2025/final20225/api.yaml)

### Детальное описание модулей (Задачи)

#### Модуль А. Архитектура приложения (Инвариант)
*   **Библиотека UIKit:**
    *   Разработать в отдельном проекте, в отдельной библиотеке.
    *   Обязательное комментирование кода (описание компонента, назначение параметров).
    *   Соблюдение принципа **DRY**.
*   **Сетевой слой (Network Layer):**
    *   Разработать в отдельном проекте, в отдельной библиотеке.
    *   Обязательное комментирование (описание методов, классов, параметров).
    *   Соблюдение **SOLID** и **DRY**.
    *   Все вызовы методов должны идти через **уровень абстракции** (чтобы библиотеку можно было заменить).
*   **Структура проекта:**
    *   Файлы распределены по папкам согласно архитектуре (DOMAIN, PRESENTATION, DATA).
    *   Изменение бизнес-логики/представления одного экрана не должно ломать другие.

#### Модуль Б. Вёрстка приложения (Инвариант)
*   **Требования к UI:**
    *   Адаптивность, отсутствие пустых пространств, отсутствие искажений.
    *   Шрифты и расстояния строго по макету.
*   **Задачи по компонентам (UIKit):**
    *   Создать проект-демонстрацию для UIKit со всеми состояниями компонентов.
    *   Реализовать: Цвета, Тему с типографикой, Иконки (шрифт), Поля ввода (все состояния), Селект (вызывает BottomSheet со смайликами), BottomSheet (высота по контенту или фулскрин со скроллом), Поиск, Кнопки, TabBar, Карточки, Хэдер.
*   **StoryBook:** Создать приложение «StoryBook» и интегрировать туда библиотеку UIKit.
*   **Экраны (Спринт-3 и 4):**
    *   SplashScreen.
    *   Логика входа: Если авторизован -> «Вход по паролю/биометрии», иначе -> «Вход и регистрация».
    *   Экран «Вход и регистрация».
    *   Экран «Создание профиля» (переход по кнопке «Зарегистрироваться»).
    *   Экран «Создание пароля» (переход по кнопке «Далее»).
    *   Главная (фильтрация категорий), Каталог (поиск, корзина), Корзина (изменение, удаление, оформление заказа с уведомлением на 5 сек), Карточка описания (BottomSheet), Список проектов, Создание проекта (фото через BottomSheet).

#### Модуль В. Клиент-серверное взаимодействие (Инвариант)
*   **Библиотека:** Создать отдельную библиотеку, содержащую:
    *   Модели от сервера.
    *   Сетевой клиент.
    *   Запросы к серверу.
    *   Обработку ошибок.
*   **Логика:**
    *   При ошибке или отсутствии сети — **вернуть ошибку**.
    *   Отображать уведомление с текстом ошибки (как на макете) в течение **5 секунд**. При нажатии — закрывается принудительно.
*   **Валидация полей:**
    *   Email: паттерн `name@domenname.ru` (только маленькие буквы и цифры).
    *   Пароль: минимум 8 символов, заглавные и строчные буквы, цифры, пробелы, спецсимволы.
*   **Список запросов для реализации:** Авторизация, Создание пользователя, Изменение профиля, Акции, Каталог, Поиск, Товар, Корзина (добавление/изм/оформление), Проекты (список/создание), Профиль, Выход.

#### Модуль Г. Хранение информации (Инвариант)
*   **Работа с Git:**
    *   Для каждого спринта создается ветка `Спринт Х`.
    *   В конце выполняется Pull Request в ветку `main` и закрывается.
    *   Проект не должен требовать разархивирования.
*   **Кэширование и персистентность:**
    *   Сохранять состояние любого последнего открытого экрана до перезапуска.
    *   Кэшировать все медиа-ресурсы.
    *   Кэшировать главную страницу.
    *   Сохранять данные входа и активную сессию после перезапуска.
    *   Сохранять состояние экрана «Создание профиля» (заполненные поля) после перезапуска.

#### Модуль Д. Взаимодействие с аппаратными средствами (Инвариант)
*   **Камера/Галерея:** На экране «Создание проекта» при нажатии на «+» выбор фото из галереи или камеры.
*   **Уведомления (Local Notifications):**
    *   Если пользователь неактивен в последнем открытом проекте более 3 минут — локальное уведомление.
    *   Если пользователь не открывал приложение более 1 минуты — уведомление (один раз до следующего входа).

#### Модуль Е. Тестирование (Вариатив)
*   **Unit/UI Тесты (UIKit):**
    *   Input ошибки (фон, обводка, текст, цвет).
    *   Select без иконки (открытие BottomSheet).
    *   Кнопка chips (status ON/OFF).
    *   Карточка Primary (status ADD/DELETE).
    *   Tabbar (фокусировка/активность).
*   **Тесты сетевого слоя:** Написать тесты на каждый из запросов (Авторизация, Профиль, Корзина и т.д.).

#### Модуль Ж. Подготовка продукта (Вариатив)
*   **Презентация:**
    *   Для разработчиков (техническая).
    *   Содержание: Схема классов, Производительность (сценарий «Создание проекта»), Архитектурные решения, Используемые/неиспользуемые библиотеки.
    *   Скрин диаграммы активности пользователя (по Яндекс.Метрике).
*   **Аналитика (Yandex Metrica):**
    *   Инициализировать метрику.
    *   Добавить события (actions) на каждое поле и отправку формы на экране «Создание проекта».
*   **Публикация (подготовка для ruStore):**
    *   Заполнить Яндекс-форму (название: фамилия латиницей + matule).
    *   Тип: Бесплатное.
    *   Загрузить: Иконку, Название, Тип, Категорию, Возраст, Описания (краткое/основное), Скриншоты всех размеров.
